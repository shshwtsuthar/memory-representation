## Matplotlib `__version__` Structure - Complete Analysis

### 1. Where `__version__` is Defined and Exposed

`__version__` is defined in `/lib/matplotlib/__init__.py` using a **module-level `__getattr__` mechanism** with the `@_api.caching_module_getattr` decorator. This creates a lazy-loaded property that only computes the version when first accessed.

**Implementation approach:**
```python
@_api.caching_module_getattr
class __getattr__:
    __version__ = property(lambda self: _get_version())
    __version_info__ = property(lambda self: _parse_to_version_info(self.__version__))
```

### 2. How Version Information is Parsed and Structured

#### `_VersionInfo` Named Tuple
Matplotlib defines a `_VersionInfo` namedtuple modeled after Python's `sys.version_info`:
```python
_VersionInfo = namedtuple('_VersionInfo',
                          'major, minor, micro, releaselevel, serial')
```

This structure contains:
- `major`: Major version number
- `minor`: Minor version number
- `micro`: Micro/patch version number
- `releaselevel`: One of 'alpha', 'beta', 'candidate', or 'final'
- `serial`: Serial number (0 for final releases, version numbers for pre-releases)

#### `_parse_to_version_info()` Function
Converts a version string into the `_VersionInfo` namedtuple using `packaging.version.parse()`. It handles:
- **Final releases**: `releaselevel='final'`, `serial=0`
- **Dev versions**: `releaselevel='alpha'`, `serial` is the dev number
- **Pre-releases (alpha/beta/rc)**: Maps pre-release identifiers and includes the pre-release number
- **Post releases**: Treated as alpha of next micro version
